## Overview

The `addAction` method adds a callback function to a conversation flow without sending a message to the user. It's useful for performing background tasks, data processing, API calls, or flow control logic.

## Method Signature

<CodeGroup>
```typescript TypeScript
addAction(
  actionProps: ActionPropertiesGeneric | CallbackFunction<P, B>,
  cb?: CallbackFunction<P, B>,
  nested?: TFlow<P, B> | TFlow<P, B>[] | null
): TFlow<P, B>
```

```javascript JavaScript
addAction(actionProps, cb, nested)
```
</CodeGroup>

## Parameters

<ParamField path="actionProps" type="ActionPropertiesGeneric | CallbackFunction">
  Either configuration options or the callback function directly.
  
  When passing options object:
  <Expandable title="properties">
    <ParamField path="capture" type="boolean" default="false">
      When `true`, captures the user's next message and passes it to the callback.
    </ParamField>
    
    <ParamField path="delay" type="number" default="0">
      Delay in milliseconds before executing the action.
    </ParamField>
    
    <ParamField path="idle" type="number" deprecated>
      Deprecated. Idle timeout in milliseconds.
    </ParamField>
    
    <ParamField path="media" type="string">
      Media URL or path (typically used with capture).
    </ParamField>
    
    <ParamField path="buttons" type="Button[]">
      Array of button objects (typically used with capture).
    </ParamField>
  </Expandable>
  
  When passing a function directly, it becomes the callback.
</ParamField>

<ParamField path="cb" type="CallbackFunction<P, B>">
  The callback function to execute (when first parameter is options object).
  
  <Expandable title="callback parameters">
    <ParamField path="ctx" type="BotContext">
      Message context containing:
      - `body`: User's message text
      - `from`: User's phone number/ID
      - `name`: User's name
      - Additional provider-specific data
    </ParamField>
    
    <ParamField path="methods" type="BotMethods">
      Bot control methods:
      - `flowDynamic(messages)`: Send dynamic messages
      - `gotoFlow(flow, step?)`: Navigate to another flow
      - `endFlow(message?)`: End the conversation
      - `fallBack(message?)`: Return to previous step
      - `state`: User-specific state management
      - `globalState`: Global bot state management
      - `provider`: Provider instance
      - `database`: Database instance
      - `blacklist`: Blacklist management
      - `queue`: Queue management
      - `extensions`: Custom extensions
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="nested" type="TFlow | TFlow[]">
  Optional child flow(s) to include in the action's scope.
</ParamField>

## Return Value

<ResponseField name="flow" type="TFlow<P, B>">
  Returns the flow object for method chaining.
</ResponseField>

### Database Operations

<Tabs>
  <Tab title="TypeScript">
    ```typescript
    const registerFlow = addKeyword('register')
      .addAnswer(
        'What is your email?',
        { capture: true }

      .addAction(async (ctx, { state, database, flowDynamic }) => {
        // Save to database
        await database.save({
          from: ctx.from,
          email: ctx.body,

    const cartFlow = addKeyword('cart')
      .addAnswer('What would you like to add?', { capture: true })
      .addAction(async (ctx, { state, flowDynamic }) => {
        // Get current cart
        const cart = state.get<string[]>('cart') || []
        
        // Add item
        cart.push(ctx.body)
        await state.update({ cart })
        
        await flowDynamic(`Added "${ctx.body}" to cart. Total items: ${cart.length}`)
      })
      .addAnswer('Add another item?', { capture: true })
      .addAction(async (ctx, { state, flowDynamic, endFlow }) => {
        if (ctx.body.toLowerCase() === 'no') {
          const cart = state.get<string[]>('cart')
          await flowDynamic(`Final cart: ${cart.join(', ')}`)
          return endFlow()
        }
      })
    ```

## Key Differences from addAnswer

| Feature | addAnswer | addAction |
|---------|-----------|----------|
| Sends message | Yes | No |
| Executes callback | Optional | Required |
| Use case | Respond to user | Background processing |
| Visibility | User sees message | Silent operation |

## Notes

- `addAction` does not send any message by default - use `flowDynamic()` to send messages from within the callback
- Perfect for data validation, API calls, database operations, and flow control
- When `capture: true`, the action waits for user input before executing
- Callbacks are async-compatible - always use `async/await` for asynchronous operations
- Use `endFlow()` to stop the conversation, `fallBack()` to return to previous step
- State changes persist across the conversation for the same user
